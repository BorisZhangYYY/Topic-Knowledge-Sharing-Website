### 2.1 website-backend

#### 接口一览

| 序号 | 路由 | 请求类型 | 响应数据 | 开发进度 |
| :---: | :--- | :---: | :--- | :--- |
| 1 | `/api/auth/register` | POST | message + user_id + username + access_token | ✅ 已完成（v0.5） |
| 2 | `/api/auth/login` | POST | message + user_id + username + access_token + needs_email_verify | ✅ 已完成（v0.5） |
| 3 | `/api/auth/logout` | POST | message + detail | ✅ 已完成（v0.5） |
| 4 | `/api/auth/email_verifying` | POST | message + code（dev模式）+ detail | ✅ 已完成（v0.5） |
| 5 | `/api/auth/reset_success` | POST | message + detail | ✅ 已完成（v0.5） |

#### 接口详细说明

**POST `/api/auth/register`**
- 请求体：`{ "username": "...", "password": "...", "email": "..."（可选） }`
- 成功响应（201）：`{ "message": "ok", "user_id": 1, "username": "...", "access_token": "..." }`
- 校验规则：
  - username：3~30位，字母开头，仅含字母/数字/下划线
  - password：≥8位，含大写字母、小写字母、数字各至少1个
  - email：标准邮箱格式（可选）
- 错误响应：400（校验失败）、409（用户名/邮箱已存在）

**POST `/api/auth/login`**
- 请求体：`{ "username": "...", "password": "..." }`
- 成功响应（200）：`{ "message": "ok", "user_id": 1, "username": "...", "access_token": "...", "needs_email_verify": true/false }`
- `needs_email_verify` 为 `true` 的条件：首次登录 或 上次登录超过7天
- 登录成功后自动更新 `last_login_at`
- 错误响应：400（参数缺失）、401（账号或密码错误）

**POST `/api/auth/logout`**
- 请求头：`Authorization: Bearer <token>`
- 成功响应（200）：`{ "message": "ok", "detail": "logged_out" }`
- Token 加入内存黑名单，立即失效
- 错误响应：401（未认证）

**POST `/api/auth/email_verifying`**
- 请求体：`{ "email": "..." }`
- 成功响应（200）：`{ "message": "ok", "code": "XXXXXX", "detail": "dev_mode" }`
- OTP 为6位数字，有效期10分钟，存储于内存
- 当前为开发模式，直接在响应中返回验证码（生产环境需接入邮件服务）
- 错误响应：400（邮箱格式错误）

**POST `/api/auth/reset_success`**
- 请求体：`{ "email": "...", "otp_code": "XXXXXX", "new_password": "..." }`
- 成功响应（200）：`{ "message": "ok", "detail": "password_reset_success" }`
- 校验顺序：OTP有效性 → 新密码强度 → 查找用户 → 新旧密码不能相同 → 更新
- 错误响应：400（校验失败/OTP无效或过期）、404（邮箱未注册）

#### 中间件

- `app/auth/middleware.py` — `require_auth` 装饰器
  - 解析 `Authorization: Bearer <token>` 请求头
  - 验证通过后将 `{ "user_id": ..., "username": ... }` 写入 `g.current_user`
  - 验证失败返回 `{ "message": "unauthorized" }` 401

#### 参数校验器（`app/auth/validation.py`）

| 函数 | 规则 |
| :--- | :--- |
| `validate_username(username)` | 3~30位，字母开头，仅含字母/数字/下划线 |
| `validate_password(password)` | ≥8位，含大写/小写/数字各≥1 |
| `validate_email(email)` | `local@domain.tld` 基本格式校验 |

### 2.2 PostgreSQL

#### 表结构

**`user_info`**（用户信息表）

| 列名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | BIGSERIAL | PRIMARY KEY | 自增主键 |
| `username` | TEXT | UNIQUE NOT NULL | 用户名 |
| `password_hash` | TEXT | NOT NULL | 密码哈希（Werkzeug PBKDF2） |
| `email` | TEXT | UNIQUE, 可空 | 邮箱（可选绑定） |
| `is_active` | BOOLEAN | NOT NULL DEFAULT TRUE | 账号是否激活 |
| `email_verified` | BOOLEAN | NOT NULL DEFAULT FALSE | 邮箱是否已验证 |
| `last_login_at` | TIMESTAMPTZ | 可空 | 上次登录时间 |
| `created_at` | TIMESTAMPTZ | NOT NULL DEFAULT NOW() | 注册时间 |

**索引**

| 索引名 | 字段 | 类型 |
| :--- | :--- | :--- |
| `user_info_pkey` | `id` | UNIQUE（主键） |
| `user_info_username_key` | `username` | UNIQUE |
| `user_info_email_key` | `email` | UNIQUE |
| `idx_user_info_username` | `username` | 普通索引 |
| `idx_user_info_email` | `email` | 普通索引 |

**`schema_migrations`**（版本迁移追踪表）

| 列名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `version` | INT | 主键，迁移版本号 |
| `applied_at` | TIMESTAMPTZ | 迁移应用时间 |

#### 版本迁移策略

迁移由 `app/db/migration.py` 中的 `run_migrations()` 函数管理：

- **Migration v1**：全新数据库建表，包含所有8个字段
- **Migration v2**：存量数据库增量 ALTER TABLE，通过 PL/pgSQL `DO $$...$$` 块幂等地添加新字段，并创建索引
- 每次迁移完成后在 `schema_migrations` 中记录版本号，下次启动自动跳过已应用版本

#### 页面组件说明

**`LoginPage.vue`**
- 用户名 + 密码输入框，登录按钮（带 loading 状态）
- 登录成功后将 `access_token`、`username` 写入 `localStorage`，跳转 `/home`
- `needs_email_verify: true` 时展示黄色提示横幅
- 底部链接：忘记密码 → `/forgot-password`，注册 → `/register`

**`Register.vue`**（已有，本次升级）
- 用户名 + 密码输入，注册后跳转登录页

**`ForgotPasswordPage.vue`**
- 3步流程 + 成功状态，步骤指示器可视化
  1. **Step 1**：输入邮箱，发送验证码
  2. **Step 2**：输入6位 OTP
  3. **Step 3**：输入新密码 + 确认密码（客户端一致性校验）
  4. **Success**：🎉 成功提示 + "返回登录" 按钮

**`HomePage.vue`**
- CSS Grid 三栏布局（侧栏 240px + 主区 1fr + 右侧面板 260px）
- 响应式：≤1024px 二栏，≤768px 单栏 + 侧栏抽屉（汉堡菜单）
- **侧栏**：用户头像（字母缩写）、用户名、导航菜单（发布/草稿箱/历史/好友/个人主页/设置）、退出登录按钮
- **顶部搜索栏**：占位输入框，标注"Coming soon"
- **主区**：🔥 知识流标题 + "v0.6敬请期待" + 3张占位文章卡片（含标签、摘要、阅读更多）
- **右侧面板**：书签列表（空状态）+ 热门话题（空状态）
